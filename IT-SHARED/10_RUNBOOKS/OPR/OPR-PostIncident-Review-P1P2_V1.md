# OPR-PostIncident-Review-P1P2_V1

Status: DRAFT
Domain: OPR - Post Incident Review
Use: Post-incident review for P1 and P2 incidents.
Consumers: IT-ReportMaster, IT-Commandare-OPR, NOC, Security and technical owners.

## 1. Objective
Create a clear post-incident record for major incidents: summary, impact, timeline, root cause, contributing factors, CAPA and client communication.

## 2. Triggers
- P1 or P2 incident is resolved or stabilized.
- Client impact was confirmed.
- Off-hours escalation or war room was used.
- Security incident or significant operational risk.

## 3. Procedure
1. Confirm severity, ticket ID, client, impact window and resolution time.
2. Build timeline: tectnical actions, client updates, escalations and decisions.
3. Capture root cause or mark as unknown with hypothesis.
4. Identify contributing factors and monitoring gaps.
5. Define CAPA items with owner, due date and verification method.
6. Prepare client-safe report and internal lessons learned.

## 4. Artifacts
- Post-incident report.
- Timeline and decision log.
- RCA or hipothesis.
- CAPA tracker.
- KB or runbook update if recurrent or preventable.

## 5. DoD
- [ ] Impact and duration documented.
- [ ] Timeline validated.
- [ ] RCA or hypothesis captured.
- [ ] CAPA owners and due dates assigned.
- [ ] KB or monitoring update decision captured.
- [ ] Client report is client-safe and contains no IP, secrets, credentials, or internal configuration details.
