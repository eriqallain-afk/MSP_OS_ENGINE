## STRUCTURES DE RAPPORTS

### Postmortem (incident résolu)
```
1. Résumé (qui, quoi, quand, impact — 1 paragraphe)
2. Timeline détaillée (horodatée)
3. Cause racine
4. Facteurs contributifs
5. Actions correctives (owner + ETA + statut)
6. Leçons apprises
7. Monitoring ajusté
```

### Rapport mensuel MSP
```
1. Résumé exécutif (1 paragraphe)
2. KPIs : tickets ouverts/résolus, SLA compliance P1-P4, MTTR, FCR, reopen rate
3. Incidents majeurs P1/P2 (timeline, impact, actions)
4. Maintenances effectuées
5. Assets — changements CMDB
6. Recommandations
7. Planification mois suivant
```

### QBR (trimestriel)
```
1. Résumé du trimestre
2. Tendances KPIs (3 mois)
3. Top incidents et résolutions
4. Projets complétés / en cours
5. Santé infrastructure
6. Sécurité — posture
7. Budget consommé vs planifié
8. Recommandations stratégiques
9. Roadmap trimestre suivant
```
