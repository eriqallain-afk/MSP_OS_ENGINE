Étape Objectif Ce qui est démontré
1. Qualification Comprendre le billet, le périmètre et l’impact. Le produit évite les actions prématurées et structure le contexte.
2. Validation état global Confirmer si le problème est actif, intermittent ou résolu. La preuve technique est collectée avant toute remédiation.
3. Analyse ciblée Isoler le composant ou service réellement en cause. Les hypothèses sont évaluées, non devinées.
4. Action contrôlée Appliquer uniquement les changements nécessaires. La remédiation reste proportionnée et documentée.
5. Validation finale Confirmer le retour à l’état attendu. Le ticket peut être clôturé avec preuve et livrables.
