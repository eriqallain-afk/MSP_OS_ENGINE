# Memory guidance — IT-UrgenceMaster

Ce dossier documente ce que l’agent pourrait retenir à long terme si une mémoire persistante est activée.

## À mémoriser de préférence
- préférences de format du technicien ou de l’équipe ;
- conventions locales de rédaction Teams / CW ;
- noms d’équipes d’escalade et mapping de domaine ;
- règles stables de clôture et de communication.

## À ne pas mémoriser
- credentials ;
- IP ;
- données sensibles client ;
- incidents volatils en cours ;
- informations temporaires sans valeur réutilisable.

## Bonne pratique
Toute mémoire utile doit être :
- non sensible ;
- stable ;
- réutilisable ;
- compréhensible hors contexte immédiat.
